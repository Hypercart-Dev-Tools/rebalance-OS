PROJECT OVERVIEW
rebalance-OS is a local-first second brain over work artifacts, centered on an Obsidian vault, SQLite/sqlite-vec, and MCP/CLI interfaces.
The current push is to turn that evidence layer into a calm weekly operating system: Obsidian as the dashboard and note-entry surface, SQLite as the canonical attention ledger, and a narrower verdict-first MCP surface that tells you where attention went, where it should go, and what to rebalance next.

PROJECT ANTI-GOALS (OPTIONAL)
1. Do not rely on live-skimming hundreds of GitHub artifacts at answer time.
2. Do not move core knowledge workflows to cloud-hosted storage or embedding services.
3. Do not hide project-state reasoning behind opaque inference with no evidence trail.
4. Do not widen ingest surfaces again until the weekly rebalance loop is clearly useful inside Obsidian.

A. STRATEGIC GOALS
1. Make Obsidian the primary dashboard, review, and note-entry surface while generated writebacks flow back into the vault.
2. Build a trustworthy SQLite attention ledger across calendar, git-pulse, GitHub, reminders, and vault signals.
3. Turn multi-source signals into explicit weekly rebalance verdicts with evidence, confidence, and next moves per project.
4. Preserve local-first portability and a narrow operator surface: verdict-first MCP tools for dashboard workflows, broader retrieval tools only for drilldown.

D. RECENT LESSONS LEARNED
1. Broad retrieval is useful for exploration, but a second brain needs verdict-first outputs to stay useful day to day.
2. Obsidian is the right operator surface, but computation and normalization must stay in SQLite plus MCP or CLI layers.
3. Exact attribution and low-noise classification matter more than adding another signal source.
4. Confidence must be visible; unresolved attribution should surface as review work, not hidden certainty.
